The shift

Code got cheap.
Judgment didn't.

When agents write most of the code, volume stops being the bottleneck. The real risk becomes drift — code that quietly diverges from the architecture and product decisions your team already made.

So review moves up the stack. The question is no longer “is this line correct?” but “does this honor the decision we made?” That judgment is the part that didn't get cheap.

How it works

Three surfaces. One source of truth.

The same governed handbook, seen from three sides — so intent, review, and execution finally point at one place.

Product & eng leaders

Declare intent

State a decision and see conflicts, gaps, and blast radius the moment it lands — before it ever reaches code.

Engineers

Review the decision, not the diff

Every pull request is checked against the handbook your team governs, so review is about judgment — not line-by-line policing.

Coding agents

Build from the same truth

Agents read the governed handbook over MCP, so what they ship inherits your team's architecture and product decisions by default.

Boundaries

What kaixn is not.

Not a coding agent
It governs the decisions your agents follow — it doesn't write the code. Bring whichever agent you like.
Not a linter
Linters catch style and syntax. kaixn guards the architecture and product judgment a linter can't see.
Not a doc store
Docs go stale in a drawer. This is a living source of truth — mined, governed, and enforced.
